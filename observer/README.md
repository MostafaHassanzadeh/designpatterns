Observer pattern
================

### Overview

Ever got notifications on available OS updates? The chances are, yes. So you
already can imagine how Observer pattern looks like in action. Simply put, one
or more objects subscribe to an observer object which notifies subscribers when
an event occurs.

### Reasons to use

If the answer to following question is "yes", then most likely you should use
this pattern:

- Is there an object that should be notified by another object when something
  occurs?
